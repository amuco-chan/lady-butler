# Lady's Butler — パーソナル執事AI MVP

大学の課題、日常タスク、相談をひとつにまとめ、止まったときに「次の10分」を示す個人用Webアプリです。

## MVP仕様

- タスクの追加・編集・削除・完了、検索、状態フィルター、締切／優先度ソート
- 締切、優先度、残り進捗、所要時間に基づく「今日やること」の自動整理
- タスク内容を参照する執事チャット（通常相談、タスク、課題、進捗モード）
- 「○○をタスクに追加して」で、締切・優先度・所要時間を読み取り自動登録
- 課題文と自分のメモから、意図・構成・下書き・提出前チェックを生成
- 設定、タスク、会話をブラウザ内に自動保存
- 5段階の気分チェックインと一言メモ
- 短い日記、執事による振り返り、過去の日記一覧
- 気分に応じた今日のタスク量調整とチャットへの日記反映
- PC／スマートフォン対応
- スマートフォン用の下部タブと、ホーム画面へ追加できるPWA構成

## 技術構成

- React + TypeScript + Vite
- CSSによるレスポンシブUI
- Lucide React（アイコン）
- localStorage（個人用MVPのデータ保存）
- ローカル執事ルールエンジン（APIキー不要）

APIキーやサーバーなしで、ダウンロード後すぐに動く構成を優先しています。LLM APIを接続する場合も、`src/lib.ts` の応答・生成関数をAPI呼び出しへ置き換えれば画面側はそのまま利用できます。

## 画面一覧

1. ホーム：今日の最優先、予定時間、締切間近、最初の10分
2. タスク：一覧、検索、並び替え、登録、編集、削除、完了
3. 執事に相談：5モードのチャットとタスクを踏まえた行動提案
4. 課題サポート：課題の読み解き、構成、下書き、チェック
5. 設定：呼び名、口調、厳しさ、通知頻度、データ削除
6. 日記・気分：気分、できたこと、しんどかったこと、明日への持ち越し、AIコメント

## データ設計

`Task`、`ChatMessage`、`MoodLog`、`DiaryEntry`、`Settings` の型を `src/types.ts` に定義しています。Taskには締切、カテゴリ、優先度、進捗、推定時間、状態、メモ、作成・更新日時を保持します。気分と日記も端末のlocalStorageに保存し、外部へ送信しません。

## 開発手順

1. 画面とデータ型を定義
2. 端末内の保存処理を追加
3. タスクCRUDと今日の優先順位ロジックを実装
4. 執事応答と課題支援を実装
5. レスポンシブUIを調整
6. TypeScriptビルドとブラウザ操作で検証

## 起動方法

```bash
npm install
npm run dev
```

動作検査は `npm test`、本番用ビルドは `npm run build` で実行できます。

表示された `http://localhost:5174` をブラウザで開きます。本番用ファイルは `npm run build` で生成できます。

## スマートフォンで使う

GitHub Pagesで公開すると、PCを起動していないときも次のURLから開けます。

`https://amuco-chan.github.io/lady-butler/`

iPhoneはSafariの共有メニューから「ホーム画面に追加」、Androidはブラウザメニューから「アプリをインストール」を選べます。

タスク・日記・会話は、開いた端末のブラウザ内に保存されます。PCとスマートフォン間の自動同期は、将来のデータベース連携で追加できます。

## 主なファイル

- `src/App.tsx` — 画面、タスク操作、フォーム
- `src/lib.ts` — 保存、優先順位、執事応答、課題生成
- `src/types.ts` — データ型
- `src/styles.css` — UIデザインとスマートフォン対応

# Lady's Butler — パーソナル執事AI MVP

日常タスク、予定、気分ログ、日記をひとつにまとめ、止まったときに「次の10分」を示す個人用Webアプリです。

## MVP仕様

- タスクの追加・編集・削除・完了、検索、状態フィルター、締切／優先度ソート
- 締切、優先度、残り進捗、所要時間に基づく「今日やること」の自動整理
- 設定、タスク、日記、気分ログをブラウザ内に自動保存
- 5段階の気分チェックインと一言メモ
- 短い日記、執事による振り返り、過去の日記一覧
- 気分に応じた今日のタスク量調整
- カレンダー画面で予定の追加・編集・削除
- 月カレンダー、毎日・毎週・毎月の繰り返し予定
- Googleカレンダーから書き出した `.ics` ファイルの読み込み
- Custom GPTから届いた内容を自動追加し、曖昧なものだけ確認
- Custom GPTが許可されたタスク・予定・気分・日記を必要時に参照
- 同じ同期キーによるPC・スマートフォン間の自動データ同期
- PC／スマートフォン対応
- スマートフォン用の下部タブと、ホーム画面へ追加できるPWA構成

## 技術構成

- React + TypeScript + Vite
- CSSによるレスポンシブUI
- Lucide React（アイコン）
- localStorage（オフラインでも使える端末内保存）
- Vercel Functions（Custom GPT連携・端末間同期API）
- Upstash Redis（GPT受信箱とアプリデータのクラウド保存）
- 執事らしい短い振り返り文のローカル生成

チャット機能と課題サポート機能はアプリから外しています。記録と提案は、タスク・予定・気分ログ・日記を中心に使います。Custom GPTから届いた内容は、明確なら自動追加し、日時などが曖昧なものだけ確認待ちに残します。

## 画面一覧

1. ホーム：今日の最優先、予定時間、締切間近、最初の10分
2. タスク：一覧、検索、並び替え、登録、編集、削除、完了
3. カレンダー：予定一覧、予定追加、編集、削除、自然文からのGPT連携案内
4. 日記・気分：気分、できたこと、しんどかったこと、明日への持ち越し、AIコメント
5. 設定：呼び名、口調、通知、PC・スマホ同期、GPT連携、バックアップ

## Custom GPTとつなぐ

Custom GPTのActionsに、次のOpenAPI URLを登録します。

`https://lady-butler.vercel.app/gpt-action-openapi.json`

GPTが課題・買い物などをタスク候補として、面談・授業・バイト・約束など時間の決まったものを予定候補として送ります。さらに相談内容に必要な時だけ `/api/gpt-context` を呼び、設定で許可した予定・やること・直近の気分・日記を提案へ反映できます。日記は長く引用せず、記録がない内容は推測しない設計です。

### GPT直接同期を有効にする

1. Vercel MarketplaceからUpstash Redisをこのプロジェクトへ接続します。
2. 別のランダム文字列を `GPT_ACTION_TOKEN` として追加します。Custom GPTのActions認証をAPI Key・Bearerにし、この値を登録します。
3. Productionへ再デプロイします。
4. Lady Butlerの「設定 → PC・スマホ同期」で「同期キーを作る」を押します。作成されたキーはクラウドへ自動登録されます。
5. スマホ側には同じ同期キーを貼り付けます。

Upstashの標準トークンはVercelの環境変数だけに保存し、ブラウザやCustom GPTには登録しません。同期キーそのものはサーバーへ保存せず、照合用のハッシュだけをクラウドに保存します。`SYNC_ACCESS_TOKEN` は手動で固定キーを使いたい時だけ任意で設定できます。GPT用キーはタスク・予定の受信と許可された記録の参照だけに使い、端末同期データの更新権限とは分離します。必要な環境変数名は `.env.example` にあります。

Custom GPTのActions認証に入れるのは、アプリ画面で作る「共通の同期キー」ではなく `GPT_ACTION_TOKEN` です。共通の同期キーはPC・スマホ同期とアプリ内の共有データ診断に使います。

プライバシー方針は `/privacy.html` で公開しています。設定画面からクラウド上の記録、GPT受信待ち、通知先を削除し、同期を解除できます。

## データ設計

`Task`、`CalendarEvent`、`MoodLog`、`DiaryEntry`、`Settings` の型を `src/types.ts` に定義しています。各データは端末のlocalStorageへ保存し、同期キー設定時は暗号化通信で個人用クラウドへも保存します。同期キーそのものは端末内に保持し、クラウド側には照合用ハッシュだけを保存します。

## 開発手順

1. 画面とデータ型を定義
2. 端末内の保存処理を追加
3. タスクCRUDと今日の優先順位ロジックを実装
4. 日記の振り返りを実装
5. レスポンシブUIを調整
6. TypeScriptビルドとブラウザ操作で検証

## 起動方法

```bash
npm install
npm run dev
```

動作検査は `npm test`、本番用ビルドは `npm run build` で実行できます。

表示された `http://localhost:5174` をブラウザで開きます。本番用ファイルは `npm run build` で生成できます。

## スマートフォンで使う

Vercelで公開しているため、PCを起動していないときも次のURLから開けます。

`https://lady-butler.vercel.app/`

iPhoneはSafariの共有メニューから「ホーム画面に追加」、Androidはブラウザメニューから「アプリをインストール」を選べます。

スマホの「設定 → PC・スマホ同期」にPCと同じ同期キーを貼り付けると、やること・予定・日記・気分・設定が自動で揃います。同期キーは設定画面の「同期キーを作る」から作成でき、クラウド側にも自動登録されます。変更は約1秒後に保存され、アプリへ戻った時にも最新状態を確認します。

## 主なファイル

- `src/App.tsx` — 画面、タスク操作、フォーム
- `src/lib.ts` — 保存、優先順位、執事応答、GPT受信データ変換
- `src/types.ts` — データ型
- `src/styles.css` — UIデザインとスマートフォン対応
- `api/gpt-inbox.js` — Custom GPT候補を確認待ちへ直接同期するAPI
- `api/gpt-context.js` — Custom GPTへ許可範囲だけ返す読み取り専用API
- `api/app-data.js` — PC・スマートフォン間のアプリデータ同期
- `public/gpt-action-openapi.json` — Custom GPT Actions用のOpenAPI定義
